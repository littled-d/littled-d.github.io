Spring

1、Spring简介

1.1、Spring概念

​ Spring是分层的JavaSE/EE应用full-stack(全栈)轻量级开源框架，以IoC(Inverse ofControl:反转控制：反转bean的控制权)和AOP(Aspect Oriented Programming:面向切面编程)为内核；

​ 提供了展现层SpringMVC和持久层Spring JDBCTemplate 以及业务层事务管理等众多的企业级应用技术，还能真和开源世界众多著名的第三方框架和类库，逐渐成为使用最多的JavaEE企业应用开源框架。

​ 设计理念：使现有的技术更加容易使用，本身就是一个大杂烩，整合了现有的学习框架；

SSH：Struct2 + Spring + Hibernate

SSM：Spring + SpringMvc + Mybaties

官网：https://spring.io/projects/spring-framework

1.2、Spring发展历程

  • 1997年，IBM提出来EJB的思想
  • 1998年，SUN制定开发标准规范EJB1.0
  • 1999年，EJB1.1发布
  • 。。。
  • 2006年，EJB3.0发布
  • RodJohnson(Spring之父)
    • 2002年:《Expert One-to-One J2EE Design and Development》阐述了J2EE使用EJB开发设计的优点
    • 2004年：《Expert One-to-One J2EE Development without EJB》阐述了JEE开发不使用EJB的解决方式(Spring雏形)
  • 2017年9月，发布了Spring的最新Spring5.0通用版(GA)

1.3、Spring的优势

  • 开源免费，轻量级，非入侵式的

  • 方便解耦，简化开发

    通过Spring提供的IoC容器，可以将对象间的依赖关系交由Spring进行控制，避免硬编码所造成的过度耦合。用户也不必再为单例测试类、属性文件解析等底层要求编写代码，更加专注于上层；（例业务层调用DAO层的对象）

  • AOP编程的支持

    通过Spring的AOP功能，方便进行面向切片编程，不宜用OOP实现的可以用AOP轻易实现；

  • 声明式事物的支持

    可以将我们从单调烦闷的事务管理代码中解放出来，通过声明式当时灵活的进行事务管理，提高开发效率；

  • 方便程序的测试

    可以利用非容器依赖的编程方式进行几乎所有的测试工作，测试不再是昂贵的操作，而是随手可做的事情；Spring继承junit

  • 方便集成各种优秀的框架

    Spring对各种优秀框架(Struts、Hibernate、Hessian、Quzrtz等)的支持。

  • 降低JavaEE API的使用难度

    Spring对JavaEE API(如JDBC、JavaMail、远程调用等)进行了封装，使这些API的使用难度大为降低；

  • Java源码是经典学习范例

    Spring的源码设计精妙、结构清晰，处处体现着对Java设计模式灵活运用以及对Java技术的高深造诣。

3、IOC理论

控制反转：将主动权从程序猿转移到用户手中；

控制反转是一种设计思想，DI（注入依赖）是实现IOC的一种方法。没有IoC的中序中，对象的创建由程序自己控制，控制反转后将对象的创建转移给第三方，获得依赖对象的方式反转；

控制反转时通过描述(xml或注解)并通过第三方去生产或者获取特定的对象的方式。在Spring中实现控制反转的时Ioc容器，其实现方法是注入依赖(Dependency Injection ,DI)

config文件

import cn.xiejp.dao.pojo.User;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

// 被Spring接管，也就是一个xml问价，作用是一样的；

    // 方法名字相当于bean标签的id属性
    // 方法的返回值相当于bean的class属性

    @Bean

    }
}

6、代理模式

代理模式是SpringAOP的底层！属于23种代理模式之一！ 【SpringAOP和ASpringMVC】

源头和目的之间添加中介，例如结婚与婚介所，

6.1、代理模式的分类

  • 静态代理
  • 动态代理

6.2、静态代理

1、角色分析：

  • 抽象角色：一般使用接口或者抽象类来的实现
  • 真实角色：被代理的角色
  • 代理角色：代理真实角色，代理真实角色后，我们一般会做一些附属操作
  • 客户：访问代理对象的人

2、代码的实现步骤：

  1. 接口

    // 租房
public interface Rent {
    void rent();
}

  2. 真实角色

    // 房东
public class Host implements Rent {
    @Override
    public void rent() {
        System.out.println("房东要出租房屋！！！");
    }
}

  3. 代理角色

    public class Proxy implements Rent{
    private Host host;

    public Proxy(){}

    public Proxy(Host host){
        this.host = host;
    }

    @Override
    public void rent() {
        host.rent();
    }
    // 代理会做一些附属操作
    // 看房
    public void seeHouse(){
        System.out.println("去看房");
    }
    // 收中介费
    public void fare(){
        System.out.println("收中介费");
    }
}

  4. 客户端访问代理角色

    public class Client {
    public static void main(String[] args) {
        // 不用代理
        // 直接找房东
        Host host = new Host();
        host.rent();

        // 添加代理模式
        // 直接找中介
        Proxy proxy = new Proxy();
        // 中介的附属操作
        proxy.seeHouse();
        proxy.fare();
        // 完成做种目的
        proxy.rent();
    }
}

3、代理模式的好处

  • 可以使真实角色的操作更加纯粹！不用去关注一些公共的业务
  • 公共业务就交给了代理角色！实现了业务的分工！
  • 公共业务发生扩展的时候，方便集中管理！

缺点：

  • 一个真实角色就会产生一个代理角色，代码量会翻倍，开发效率变低；

6.3、动态代理

  • 动态代理和静态代理角色一样
  • 动态代理的代理类是动态生成的，不是直接写好的！
  • 动态代理分为类
    • 基于接口——JDK动态代理
    • 基于类—— cglib
    • java字节码实现——JAVAssist
  • 需要了解两个类
    • Proxy 代理类
    • InvocationHandler 是由代理实例的调用处理程序实现的接口，每个代理实例都有一个关联的调用
      处理程序；当代理实例上调用方法时，方法调用将被编码并分派到起调用处理程序的invoke方法；
  • 动态代理视频

1、代码实现

  1. 动态代理类模板：

    	import org.springframework.cglib.proxy.InvocationHandler;
	import org.springframework.cglib.proxy.Proxy;
	import java.lang.reflect.Method;
	
	public class ProxyInvocationHandle implements InvocationHandler {
	    // 被代理的接口
	    private Object target;
	
	    public void setTarget(Object target) {
	        this.target = target;
	    }
	
	    // 生成得到代理类
	    public Object getProxy() {
	        return Proxy.newProxyInstance(this.getClass().getClassLoader(),
	                target.getClass().getInterfaces(), this);
	    }
	
	    // 处理代理实例，并返回结果
	    @Override
	    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
	        Object result = method.invoke(target, args);
	        // 在这儿可以执行一些附属操作
	        return result;
	    }
	    // 在这儿可以添加一些附属操作
	}

2. 模板的使用：

	```java
	public class Client {
	    public static void main(String[] args) {
	        // 动态代理
	        // 真实角色
	        Host host = new Host();
	        // 代理角色 暂时不存在
	        ProxyInvocationHandle pih = new ProxyInvocationHandle();
	        // 设置要代理的对象
	        pih.setTarget(host);
	        // 动态生成代理类
	        Host proxy = (Host) pih.getProxy();
	        // 代理类执行操作
	        proxy.rent();
	    }
	}

2、动态代理的好处

  • 可以使真实角色的操作更加纯粹！不用去关注一些公共的业务
  • 公共业务就交给了代理角色！实现了业务的分工！
  • 公共业务发生扩展的时候，方便集中管理！
  • 一个动态代理类代理的是一个接口，一般就是对应的一类业务
  • 一个动态代理类可以代理多个类，只要实现了同一个接口即可

7、AOP

7.1、什么是AOP

AOP为Aspect Oriented Programming的缩写，意为：面向切面编程，通过预编译方式和运行期间动态代理实现程序功能的统一维护的一种技术。AOP是OOP的延续，是软件开发中的一个热点，也是Spring框架中的一个重要内容，是函数式编程的一种衍生范型。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

7.2、AOP在Spring中的作用

提供声明式事务；允许用户自定义切面

  • 横切关注点:跨越应用程序多个模块的方法或功能.既是,与我们业务逻辑无关,但是我们需要关注的部分,就是横切关注点.如日志,安全,缓存,事务等…
  • 切面（ASPECT）：横切关注点 被模块化 的特殊对象。即，它是一个类。
  • 通知（Advice）：切面必须要完成的工作。即，它是类中的一个方法。
  • 目标（Target）：被通知对象。
  • 代理（Proxy）：向目标对象应用通知之后创建的对象。
  • 切入点（PointCut）：切面通知 执行的 “地点”的定义。切入点简单语法 切入点详细语法 切入点表达式
  • 连接点（JointPoint）：与切入点匹配的执行点。

    public class BeforeLog implements MethodBeforeAdvice {

    /**
     * @param method 要执行的方法
     * @param args 方法的参数
     * @param target 目标对象
     */

    public void before(Method method, Object[] args, Object target) throws Throwable {

    }
}
    import org.springframework.aop.AfterReturningAdvice;
import java.lang.reflect.Method;

public class AfterLog implements AfterReturningAdvice{
    /**
     * @param returnValue 执行方法的返回值
     * @param method 要执行的方法
     * @param args 方法的参数
     * @param target 目标对象
     */
    @Override
    public void afterReturning(Object returnValue, Method method, Object[] args, Object target) throws Throwable {
        System.out.println("执行了"+method.getName()+"方法，返回结果为"+returnValue);
    }
}

    <bean id="beforeLog" class="cn.xiejp.AOPDemo.log.BeforeLog"/>

    <!--方式一-->
    <!--配置AOP，需要导入AOP的约束-->
    <aop:config>
        <!--切入点：expression：表达式；execution(要执行的位置， 修饰词，返回值，类名，方法名，参数)-->
        <!--在这儿，我们要插入的位置是StudentServiceImpl这个类，这个类中的**表示所有方法；（..）表示可以有任意个参数-->
        <aop:pointcut id="pointcut" expression="execution(* cn.xiejp.AOPDemo.Impl.StudentServiceImpl.*(..))"/>

        <!--执行环绕增加-->
        <aop:advisor advice-ref="afterLog" pointcut-ref="pointcut"/>
        <aop:advisor advice-ref="beforeLog" pointcut-ref="pointcut"/>
    </aop:config>
</beans>

方式三：使用注解实现；

  1. 使用注解定义类：

    import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;

@Aspect  // 标注这个类是切面
public class AnnotationPointCut {
    
    @Before("execution(* cn.xiejp.AOPDemo.Impl.StudentServiceImpl.*(..))")
    public void before() {
        System.out.println("----------方法前-----------");
    }
    
    @After("execution(* cn.xiejp.AOPDemo.Impl.StudentServiceImpl.*(..))")
    public void after() {
        System.out.println("----------方法后-----------");
    }
    // 环绕增强中,可以给定一个参数,代表要获取处理的点"
    @Around("execution(* cn.xiejp.AOPDemo.Impl.StudentServiceImpl.*(..))")
    public void around(ProceedingJoinPoint jp) {
        System.out.println("===环绕前===");
        System.out.println(jp.getSignature());  // 获得签名并打印
        try {
            Object proceed = jp.proceed();  // 执行方法并得到返回值
            System.out.println(proceed);
        } catch (Throwable e) {
            e.printStackTrace();
        }
        System.out.println("===环绕后===");
    }
}

  2. 配置applicaionContext.xml文件

    <!--方式三-->
<bean id="annotationPointCut" class="cn.xiejp.AOPDemo.annotation.AnnotationPointCut"/>
<!--开启注解支持 JDK(默认 proxy-target-class="false")  cglib(设置为true即可)  -->
<aop:aspectj-autoproxy proxy-target-class="false"/>

9.1、事务

  • 把一组业务当作一个业务来做，要么都成功，要么都失败！
  • 事务在项目开发中，十分的重要，涉及到数据一致性问题；
  • 确保完整性和一致性；
  • 事务ACID原则：原子性（atomicity，或称不可分割性）、一致性（consistency）、隔离性（isolation，又称独立性）、持久性（durability）

9.2、Spring中的事务管理

  • 声明式事务：AOP
  • 编程性事务：需要在代码中进行事务的管理；

9.3、为什么需要事务？

  • 如果不配置事务，可能存在数据提交不一致情况；
  • 如果我们不在Spring中配置声明式事务，我们就需要在代码中手动配置事务；
  • 事务在项目的开发中十分重要，涉及到数据的一致性和完整性；
